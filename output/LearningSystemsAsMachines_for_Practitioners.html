Learning Systems as Abstract Machines

A short, technical introduction to a complexity-theoretic foundation for the security, privacy, and accuracy of statistical learning — from linear regression to LLMs and agent stacks. A companion to, and foundation for, the AIOracle framework.

1. The one idea

Modern machine learning is studied with a different object for every question: a function on a metric space for evasion attacks, a tamperable compiler for poisoning, a black box with a privacy budget for membership inference. Each switch of object hides the fact that they are the same system seen from different sides. This note develops a single object and reads accuracy, security, and privacy off of it.

The object is one universal machine U. A source Π is a program for U fed uniform random bits — it is what “the data” is. A learner L reads the source through bounded access and synthesizes an advice string w. Inference is U running the advice: Mw(x) = U(w, x) for teval steps. So a “model” is a time-bounded universal computation, and linear regression and an LLM are the same U-program at two different coordinates — a tiny (|w|, teval) for the regression, an enormous one for the LLM. The thing that differs is scale, not kind.

Slogan. Inference computes almost nothing — it is advice evaluation. The real computation is the learner synthesizing advice behaviourally equivalent to the source under bounded access. Learning is program synthesis; security is the hardness of controlling what advice that synthesis emits.

2. The discipline that keeps this complexity, not statistics

The standard move in learning theory is to fix a data distribution D and prove things in expectation over it. That quietly smuggles the analyst’s assumptions into the foundation. We forbid it: the distribution is demoted from a foundation to a parameter. Nothing in the primitives may name a D. The only measure in the theory is the fixed uniform measure on U’s random bits; any “data distribution” is the derived pushforward D = Π*(Uniform).

This is the same discipline cryptography learned from Shannon to Goldwasser–Micali, and the same instinct behind the generic-group model and the AIOracle’s Generic Language Model: forbid the smuggled prior, force reliance on the metered object. A consequence with teeth: two sources with the same output distribution D but different generative complexity are different objects. That difference is precisely where a statistical–computational gap can live, and it is invisible to any theory that works over D alone.

3. One quantity in five roles: Kt

The technical spine is a single complexity measure, Levin’s time-bounded Kolmogorov complexity Kt(x) = minp { |p| + log t(p) } over programs p for U. It is not chosen by taste; a conservation law forces it (a potential that composes must charge log t, not t). The same Kt then plays five roles:

RoleWhat it measures
Behavioural metricdistance between a model and the source, weighted by 2−Kt(x) over the disagreement set
Potential / conservationΦ = |w| + log ttrain + log teval, additive under reduction
CapacityKt(w), the effective complexity of the model — not raw size |w|
Security objectfinding a cheap adversarial input = finding low-Kt witnesses; OWF-equivalent (Liu–Pass)
Learner objectivethe learner minimizes Kt(w) subject to fit (Levin search)

That capacity is Kt(w) and not |w| resolves the overparameterization puzzle directly: large models generalize because the advice SGD finds is compressible (Kt(w) ≪ |w|), while robustness lower bounds genuinely need large raw |w|. Generalization is governed by Kt(w); robustness by |w|; both hold at once because they meter different resources.

4. One functional, four corners: accuracy, security, privacy

Every player — honest learner or adversary — is the same kind of object: a resource-bounded transducer with a capability vector κ (which tapes it can touch) and a budget. Define one quantity,

Risk*(T, κ, r) = the best expected loss on task T achievable by any player with capabilities κ and budget r.

Reading this one functional with two knobs — an aggregation operator ⊕ and a base — produces the entire security/privacy vocabulary as corners of one object:

CornerReadingThis is…
Accuracy⊕ = E (average), honest taskcompleteness / utility
Detection⊕ = quantiledistribution-shift / auditing
Security⊕ = sup (worst case)robustness / evasion
Privacydistinguishing on the neighbourhood basemembership inference / leakage
The central identification. Accuracy and security are one surface seen from two sides. Accuracy is the region a bounded learner can reach (a positive theory). Security is a learning lower bound — the attack task is un-learnable by a κ-bounded adversary. “Security follows from negative results” stops being a slogan and becomes a definition: a security theorem is a hardness-of-learning theorem for the attack task.

Note this replaces the metric-budget convention (“adversary constrained to an ε-ball”) with a computational budget: what the adversary can spend to find a bad input, not how far that input sits from a benign one. The capability vector κ replaces the attack taxonomy — evasion, poisoning, and backdoor are not separate categories but different settings of κ, exactly the AIOracle’s ATK ⊆ {white_box, black_box, inject_learn, inject_infer}.

5. The staircase: a graded security taxonomy with a provability ladder

Because security is un-learnability, the strength of a security guarantee is graded by how hard the corresponding learning problem is. This produces a staircase of thresholds, τstat ≤ τdetect ≤ τrecover, cutting the resource plane into regimes that replace the ad-hoc evasion/poisoning/backdoor taxonomy:

RegimeMeaningReplaces
below τstatinformation-theoretically hiddenstatistical / perfect security
τstat to τdetectpresent but undetectableundetectable backdoors; computational privacy
τdetect to τrecoverdetectable, not recoverablewatermarking; auditable-but-private
above τrecoverfully learnableaccuracy

Remarkably, the difficulty of proving security at each rung climbs through cryptography’s own provability ladder — and the cost of certifying it, the U-dependence of the bound, and the cost of zero-knowledge certification all climb together:

Provability ladder. unconditional (statistical dimension / SQ floor) → idealized-model (random-oracle / generic-group / GLM) → standard-assumption (one-way functions, via Kt-hardness) → open (Gap-Kt, the detection band).

The SQ floor is the jewel: it is the one place this theory proves security unconditionally — without betting on P ≠ NP or one-way functions — via statistical-dimension lower bounds. The idealized-model rung is where the AIOracle’s utility–privacy theorem lives; it carries the standard caveat that idealized-model proofs can be uninstantiable (Canetti–Goldreich–Halevi), so provability there slightly outruns real-world soundness.

6. The keystone: one theorem the framework genuinely owns

Most of the staircase re-reads known hardness (SQ, low-degree, planted clique, one-way functions) in a unified vocabulary. One result is native to the framework:

Keystone (universal = generalizing). The learner that minimizes Kt(w) subject to behavioural fit is simultaneously universal (it is Levin search over advice) and generalizing (low Kt(w) gives an Occam bound). These are two readings of one principle — minimum-description behavioural fit — under a single hypothesis: I ≪ s, the learner used far less information than it saw.

The bridge is Kt(w) ≲ Kt(L) + I + log t: the advice complexity is the (constant) learner-program complexity plus the information I actually pulled from the source. So “it generalizes” and “it is the canonical learner” are the same statement, and the hypothesis I ≪ s is at once Occam compression and the non-memorization condition.

7. The membrane: the organizing principle for resources, composition, and agents

One structural distinction organizes the entire subject — the membrane between what happens at training and what happens at deployment. The learner can query the source (it spends samples); inference is source-blind (it sees only w and the input). This single line sorts everything:

AspectTraining sideDeployment side
Computettrain (with source access)Φdeploy = |w| + log teval
Informationforward flow I (learning)reverse flow I← (steering)
RandomnessHtrain (privacy / DP)Heval (certified robustness / smoothing)
CompositionΛtrain (model collapse)Λeval (agent stacks)

Two compute trades are free within the deployment side (memorize vs recompute, |w| ↔ teval; trained-in vs prompted-in knowledge, weights vs context). What is not free is crossing the membrane — and that non-freeness is exactly the sample/information axis a complexity theory must keep separate from compute.

One modulus recurs across the membrane: a Lipschitz sensitivity Λ that is privacy is composability is performative stability. It obeys a clean ordering — a private learner is automatically composable and generalizing; the converse fails — which is the complexity-native form of the adaptive-data-analysis theorem (DP ⇒ stability ⇒ generalization).

8. What the framework explains, predicts, and proposes

Explains (unifies known phenomena)

  • “SOTA accuracy, trivially fooled.” Accuracy is the E-corner, security the sup-corner of one functional; a low average sensitivity Λ says nothing about the worst-case sensitivity Λsup an adversarial input excites. Functionality never implies robustness.
  • Overparameterization is not a paradox. Generalization meters Kt(w), robustness meters |w|; their gap is implicit regularization.
  • Why backdoor detection keeps losing. Undetectable backdoors sit on the recovery band (OWF-equivalent); “scan the weights” is cryptographically hard, not merely unsolved.
  • Why DP costs accuracy, and why the same noise gives certified robustness. One randomness budget HL = Htrain + Heval feeds both privacy and robustness, on opposite membrane sides.
  • Model collapse on synthetic data. Training-side composition with Λtrain > 1; a sharp phase boundary at Λ = 1.

Predicts (new, falsifiable)

  • The comp–stat envelope. The data-scaling exponent and the parameters-vs-test-time-compute exponent obey a one-sided bound θcomp ≤ Φ(θdata): no task should have cheap params↔compute tradeability with poor data-scaling. A single point above the envelope falsifies the unified Kt-axis — and since the envelope is derived from the keystone, this single experiment is a direct test of the keystone.
  • The memorization corner. On tasks with labels independent of inputs, test-time compute cannot substitute for parameters — chain-of-thought gives near-zero benefit on pure memorization but real benefit on structured reasoning. A cheap experiment.
  • Backdoor reproduction number. In a system retrained on its own traces, a planted backdoor persists across generations iff R = Λtrain · ΛC · Λevalsup · ρhit > 1. A backdoor cannot be both self-amplifying (high ρhit) and stealthy (low ρhit) — they trade off, so the dangerous regime is exactly the open detection band.

Proposes (things to build)

  • A noiseless, computational leakage metric: |w| − Kt(w). A model’s compressibility bounds how much individuating information it can have memorized — the privacy notion that hides identity by computation, not by DP’s information-destroying noise.
  • Deepest-first robustness budgeting for agent stacks: one ill-conditioned deep tool (Λevalsup ≫ 1) re-weights every commitment above it, so spend certified-robustness budget on the most-deeply-called components first.
  • A containment monitor for self-improving systems: hold R < 1 by certifying inference sensitivity or filtering traces.
  • Provenance over detection for the model supply chain: detecting a backdoor is OWF-hard, but proving a component ran a declared clean training process is an ∃/NP statement that composes via recursive proofs (proof-carrying data). Attest the curation step, do not scan the weights.

9. Relationship to the AIOracle framework

This theory and the AIOracle framework are two layers of one theory, joined at a single seam: the AIOracle’s idealized predicate φ(prompt, context, result) corresponds to a task T approximated by a U-bounded judge with budget rjudge. The AIOracle is the interface — name φ, decompose it, measure ε, publish a falsifiable claim. This note is the foundation — which claims are achievable, which are impossible, and why.

The integration is genuinely two-way — each framework sharpens the other:

Three-term composition law. When AIOracles compose, the system breach is governed by three terms of three different types: a breach probability ε (additive, union bound), a predicate-coverage mismatch e (additive, the AIOracle’s contribution), and an error-magnitude amplification Λ (multiplicative, this note’s contribution). The AIOracle’s additive bound is correct only in the contractive regime Λsup ≤ 1; above it, the additive bound silently under-counts — the model-collapse / supply-chain regime. Conversely, the coverage term e is a real cost this note’s composition theory lacked. Neither framework had all three.

The AIOracle’s GENERATE_DATA box, which we had folded away, turns out to be a first-class curation transducer C sitting between source and corpus — and it is exactly the control point of the reproduction number R (a good filter lowers ρhit; a corrupted one raises it). In an agent stack, one agent’s GENERATE_DATA is the next agent’s INFER composed with C, so a single AIOracle’s composition theorem, applied recursively, already bounds agent-stack security.

Finally, the integration answers a question the AIOracle note explicitly leaves open. Citing Bellare–Kohno on related-key attacks, it asks: for which predicates is ε-secure approximation even possible? The staircase is that catalogue: ε-secure approximation of φi is unconditional below τstat, idealized-model-provable at the GLM rung, OWF-relative in the recovery band, and open in the detection band — indexed by the source complexity Kt(Πi) and the judge budget rjudge.

10. A frontier the integration generated: intent-conditioned authorization

The AIOracle decomposition lists action-authorization (φallow) as clean and goal-fidelity (φintent) as open, separately. The interesting object is their conjunction — “is this action permitted given the user’s goal?” — which neither framework alone handles. Its complexity is Kt(φallow ∧ φintent) ≈ Kt(Πintent), the complexity of the user’s true-goal program, and it reduces to intent inference — the formal core of the scalable-oversight / value-specification problem.

Design law. An agent is either intent-blind (drop ∧ φintent → static coarse authorization, SQ-floor, certifiable) or intent-elicited (ask the user for the goal → low-complexity lookup, cost pushed to the human). It cannot cheaply infer the intent and condition on it — that path costs Kt(Πintent). Production agents fall back to coarse static policies not from laziness but because it is the only certifiable option below Kt(Πintent).

11. Honest status

This is an organizing theory, not yet a validated one. To be precise about what is and is not established:

  • Proved: the keystone (universal = generalizing under I ≪ s), plus faithful re-readings of statistical-query, low-degree, planted-clique, and one-way-function hardness in the unified vocabulary.
  • Open native conjecture: the smooth |w| ↔ teval tradeoff for natural compressible tasks — a genuine capacity-vs-inference-compute lower bound the borrowed tools cannot even state, but which appears to be as hard as a circuit lower bound.
  • Testable prediction, unmeasured: the comp–stat envelope on scaling exponents. Its experiment is a direct test of the keystone.
  • Methodological: the discipline of keeping the statistical and computational axes separate repeatedly forbids the field’s most tempting false unifications — evidence the framework has content, since a vacuous reframing forbids nothing.

The honest one-line self-classification: a genuine organizing theory with one proved keystone, one open native conjecture, one keystone-derived testable prediction, and a working forbidding-principle — built on a faithful re-reading of complexity-theoretic hardness, and joined to the AIOracle framework as its foundation.
